1. 微妙区别

1. any和unknown

any是所有类型的顶级类型，任何类型都可以赋值给any类型的值。可出现在左，可出现在右，用了跟没用没什么区别。

unknown的值只能赋值给unknown和any的值，也是任何类型都可以赋值给unknown。

  1. 出现在右边，左边只能是unknown或者any；
  2. 出现在左边，右边可以是任意类型

2. {} vs object vs Object

object代表常规的JavaScript对象，即非基础数据类型。（函数呢？）

{}代表非null，非undefined的任意类型。

Object类似{}，但要求类型不修改Object.prototype上的方法。

范围 object< Object < {}

对对象属性无要求的，用object。Object如果修改了内置方法，是会报错的。{}非null非undefined就行。

3. type和interface

type类似JS里的const，有块级作用域， 用来声明类型，可以理解为“类型变量”。多次声明同名变量的后果是会被覆盖。可以是单个类型，也可以是对象类型。声明对象时不支持声明合并

interface自带extends，多次声明同名interface，会整合成一个interface。只能是对象类型。（支持声明合并）
